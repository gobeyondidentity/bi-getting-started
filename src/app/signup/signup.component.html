Create Passkey (in line)

This is the inline create passkey flow. All a user needs to do is enter a username, and a passkey is created.

This is what happens when you enter a username and click Create Passkey
  1. A Create Identity request is made to the Beyond Identity directory (and is successful).
  2. A Create Credential Binding Job request is made to the Beyond Identity directory. The response returns a credential binding link
  3. The credential binding link is passed to the Beyond Identity SDK
  4. Upon success a private key is created in the device's hardware trust module and public key sent to Beyond Identity's cloud.
