Sign up Flow

This is the inline signup flow. All a user needs to do is enter a username, and a passkey is created.

This is what happens when you enter a username and click signup
  1. A Create Identity request is made to the Beyond Identity directory (and is successful).
  2. A Create Credential Binding Job request is made to the Beyond Identity directory. The response returns a credential binding link
  3. The credential binding link is passed to the embedded sdk
  4. During the credential binding flow the embedded SDK asks the Beyond Identity policy engine if the credential binding is allowed.
  5. If allowed a passkey will be created on the device, and can be used for future authentications
